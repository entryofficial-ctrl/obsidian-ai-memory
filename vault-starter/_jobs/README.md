# Jobs — 반복 작업 마스터 노트

반복 작업 하나당 노트 하나. 규칙은 단순하다: **"노트 하나를 읽으면 그 작업의 스킬 전체가 된다."**

각 Job 노트는 [[Templates/job|job 템플릿]]을 따라 네 부분으로 구성된다.

1. **부트 체인** — 이 작업에 필요한 노트만 정확히 링크 (볼트 전체를 읽지 않는다)
2. **절차** — 단계별 실행 방법
3. **품질 기준** — 사용자에게 보여주기 전 점검 목록
4. **Lessons** — 교정받을 때마다 접어넣는 교훈. 이것이 핵심이다: 같은 작업이 매 실행마다 나아진다

## 운영 규칙 (AI용)

- 같은 작업을 두 번째 설명받고 있다면, 그 자리에서 Job 노트를 만든다.
- 사용자가 Job의 결과물을 교정하면, 같은 턴에서 그 Job의 Lessons에 교훈을 추가한다.
- 프로젝트 전용 Job은 `projects/<프로젝트>/jobs/`에, 프로젝트를 가로지르는 Job은 여기 `_jobs/`에 둔다.
- 새 Job을 만들면 이 README의 목록에 한 줄 추가한다.
